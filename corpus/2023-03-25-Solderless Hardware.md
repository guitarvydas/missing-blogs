# 2023-03-25-Solderless HardwareIn my mind, software is just solderless hardware.  

The Big Difference between hardware components ("chips") and software components ("libraries", "objects", "functions") is that chips are inherently asynchronous.  Software is, OTOH, inherently synchronous (Call/Return).  Again, a subtle difference, but a wholly different approach to solving problems.

Ask an EE to design a solution to your problem while only using hardware chips.   If you slap the EE hard enough whenever he tries to resort to software-think, he'll come up with a completely different design.  

Then, just code that solution up in a programming language of your choice (having closures and queue classes will help).
