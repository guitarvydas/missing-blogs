# 2023-04-02-VPLs Visual Programming Languages# VPLs

I don't mean that VPLs[^vpl] should be text+*images*, but text+*technical drawings.*

[^vpl]: VPL == Visual Programming Language.

Textual programming languages cull prose down to a handful of words that can be parsed easily by machines.

E.g. "if then else" are English words, but do not constitue the full English language.  We can't parse "War and Peace" and create machine code from it, we can only parse a few words and phrases.

IMM (in my mind), technical diagrams are kinda the same.  We can cull a few diagrammatic concepts from visual art and parse them.  We can parse rectangles, ellipses and arrows.  We don't need to parse all of Rembrandt's work.

EEs already have VPLs.  They have a language of schematics, that contains only a few visual symbols.  

In the past, EE schematics used to consist of many different kinds of symbols, but, now, the VPL syntax has been reduced to only a handful of symbols - rectangles, names, wires, ground and power.  Too many different symbols produced VPLs that were "too busy" and unreadable.
