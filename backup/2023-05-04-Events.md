# 2023-05-04-EventsIn electronics, some circuits can be triggered by edges.  I happen to call these *events* in 0D.  An *event* is a pulse that signifies that something happened, but otherwise contains no further information.  

*Events* are even "smaller/thinner" than Booleans.  In electronics parlance, *events* are equivalent to "edge triggered" signals whereas Booleans would be "level triggered" signals.  

In electronics, one makes a distinction between "leading edge" and "trailing edge" triggers.  In 0D, I ignore this distinction - *events* do not have edges, *events* are 0-width blips which can occur and cause triggering.  

A consequence of this kind of thinking leads to using "big" lumps of data as *events*, say a string processor outputs a string.  The same string can be sent into more than one input - one expecting an *event* and another expecting actual data.  The same string serves two purposes - (1) an *event* signifying that it has come into existence and (2) the actual information about its contents.
