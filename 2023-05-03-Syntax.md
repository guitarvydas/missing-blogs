# 2023-05-03-Syntax
Most of current language syntax ugliness is due to accidental complexity caused by premature optimization.

Furthermore, the CFG meme inhibits an important kind of parsing that could be used for syntactic composition.  CFGs cannot parse matched pairs of brackets.  PEGs can.

PEG allows breaking free of the CFG meme and of premature optimization (by re-allowing backtracking), thus, enabing completely new syntaxes and parsing techniques, including macros for languages other than Lisp (and - shock - diagrams as syntax).  Ohm-JS is the best PEG that I know of.

## Examples of Premature Optimization

"OMG, if we allow the use of Early's parser, we might need more that 640K of RAM"

"OMG, if we allow compilers to use more than one pass, we might need more than 640K of RAM and the tool might run slower.  Instead programmers should be made to suffer with declaration before use".

"OMG, if we use anything but text, we might be forced to use computers instead of clay tablets and papyrus".

"The only way to build a big system is to allow for big source code, instead of tiny layers".  Tiny, decoupled layers make O(N^3) inconsequential.
