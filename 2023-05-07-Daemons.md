# 2023-05-07-Daemons# Daemons
All of our programming languages are LIFO based (callstack). 

To write decoupled daemons, use FIFOs and message-passing. 

Structured message-passing - IMO - is to arrange nodes in a tree. Send 'commands' (requests) Down. Receive filtered info Up.  Arranged like an efficient business - Departments with clear reporting structure, no "going over the boss's head" on the way up and no "micro-management" on the way down. Org Charts.

Building FIFOs is easy. Building FIFO-based systems is easy, but is not directly supported by function-call syntax, you have to pass messages explicitly because it ain't built into the language(s). If you have anonymous functions (or closures), then it gets even easier. 

This ain't Rocket Science, everyone discovers it on their own, but, AFAICT it ain't built into any language(s).
